## 📖 前言

### 2026 年 2 月 18 日

> [!NOTE]
> 归档时，零音并未对下文做出任何修改。

这份 README 是 Codex 写的，不过这一段是零音亲自写的。作为 VOCAEND 的出题人，hzlt!Game 2026 的主办者，Project Hazelita 的管理员，其实零音自一开始就意识到 hzlt!Game 2026 应该没啥人气了。毕竟连主办方的零音也是 CTF 的新手，不过能让 Project Hazelita 社群中至少一部分人感受到欢乐，让出题人感觉到出爽了，其实目的已经达到了——所以即使服务器被提前关停（按量计费的服务器付不起米了），hzlt!Game 最终活跃人数远少于预期，其实零音还是蛮欣慰的。

零音偶尔、或者说经常，应该说是非常鄙夷那些所谓的老资历——零音并非讨厌有能力的人，而是那些让自己看起来很权威的人，或者说讨厌官僚主义——尤其是顶着所谓对技术的热爱的旗号来进行的官僚主义。零音向来都是讨厌这一点的——顶着自嘲的帽子便能对人傲慢，将人情味的社团变成冷冰冰的技术沙盒，将正常的学习定义为「传承/继承」，将管理体制塑造为逆向淘汰的血汗工厂。是的，零音自始至终都认为兴趣应当是自驱的而非督促的，一旦用 KPI 来管理爱好者们的兴趣，兴趣就变成了劳动。毕竟，Linux 社区从来不会因为 Linus Torvalds 一个月没提交代码就要清退他。而所谓的小团体却又将新人和老人划分为封闭的圈子，老人们通过独特的黑话来制造回忆，又与新人构建起一道社交防火墙——从而新人变成了老人们谈资的观众和耗材。

零音从未觉得最僵化的行政手段可以管理最自由的黑客精神。这也是为什么零音在 Project Hazelita 社群中始终强调共情与引导，而非控制与恐惧。零音讨厌说教、讨厌趾高气扬、讨厌「你完全没经历过我的人生，却在对我指手画脚」。hzlt!Game 2026 的一部分初衷源于此，但更多还是希望陪大家一起好好跨年。虽然零音为了这个 7 天的比赛忙了 20 天，高数作业一笔未动（不过期末还是考了 96 哈哈哈），但零音觉得很有意义就好了。至于会不会有 h!G27，一切看运气吧。

最后，VOCAEND 这首歌真的是神曲，大伙一定要听啊😭`data/` 文件夹下有社群群友想要对彼此说的话。大概就是这么多。

> Even if whole world fades away, I won't stop singing my heart out.

### 2026 年 7 月 9 日

其实零音从来没有想过再提起这些事。

如果不是最近在重构所有的 Repo，零音估计都要把 VOCAEND 忘了。返回来看了看自己之前写的代码，写得真的不尽人意啊——很多地方都极为离谱诡谲。但毕竟当时整个题都是赶工出来的，也情有可原。毕竟当时的零音，碰 C 语言还不到 4 个月呢。现在的零音有了稍微强一点的技术， ~~但主要还是有了更强的 Agent~~ 干脆直接又优化了整个项目。

不过，盯着 VS Code 的黑白屏幕三四个小时，从代码的海洋中抽身后，打开 README，重又看了看自己 2 月份写的「前言」，再回想起这 4 个月里经历的事情，于是沉默的零音更沉默了。零音费尽浑身的力气，奋力游到的大学啊，并没有零音想象中的那么完善——这里当然没有高三的紧绷，每一节课都要打足鸡血；但被各种没有意义的水课和各种神秘莫测的事情浸泡过后，却也疲惫了不少。尤其是因为下学期天天翘课和上课纯玩的缘故，经历了一个并不令人舒适的两周期末周后，零音一整只小水母，到现在都没有完全恢复过来。偶尔当然会心有余悸。

hzlt!Game 的相关事宜，在 2 月的前言中就已经阐明。零音的朋友们很多次向零音建议多做宣发，可是零音完全没意识到宣发的必要性，导致比赛都结束了后，甚至 Hazelita 社群里的很多朋友都不知道有这场比赛。平台账号注册了大约 50 个，不过真正参赛的只有 6 个。服务器是按量计费的，不过尚且在零音的预算范围内，只是那段时间零音重又精神失常，被双相搞得心力交瘁了，于是就关掉服务器了。甚至连题目镜像都没留下哪怕一个。

老实说，自高二抑郁以来，零音的精神状态一直是如此，喜怒无常，甚至在高三后期转成了中度抑郁和重度焦虑。虽然平时没有察觉，但上了大学后，这种状态真的改善了很多。于零音而言，零音不愿意说「活着是最幸福的事」这种听起来就很不零音的言语——但活着的确是幸运的事。至少正如零音一直认为的，活着便可能有生的希望——希望，本质是可能性的量度。但零音并没有宣发好 hzlt!Game，有些遗憾。这场比赛中有很多很有意思的题目，事实上质量很高的同时也好玩，比某些比赛纯粹出高难题目恶心选手好多了。当然，你可以辩论道这些 CTF 比赛有其自己的，出于区分度和赛事本身性质的考量——而零音也本就赞成高难题目，但当高难从难度滑到诡异程度和怪异程度之后，题目便可能失了一些它真正应该具有的东西了。

那群将零音捧上神坛的人——最终还是遗忘了零音。有人说其实他们并没有遗忘零音，是零音自己选择了被忘却的宿命。啊，宿命！零音固然是知道的。毕竟主动退坑是零音提的，渐渐疏远是零音选的；那照此这么说，零音的一切焦虑、抑郁、紧张和一次次失落、失望、被压制而无法反抗，便都是零音自己选择的吗？零音本不相信宿命——零音也相信以至于信奉「我命由我不由天」，即使零音对《哪吒 2》本身没什么好感。有些东西是纯粹的恶，大家都能感觉得到，却也心知肚明。但有些东西，本质上是恶，却并不纯粹——如同棒棒糖里藏了一片会自动溶解的刀片——你吃完了一整个棒棒糖，刀片也已消失不见，但你的嘴却已鲜血淋漓。这便是零音想说的和要说的了。

混沌的世界依然存在！零音蜷缩于 LyRiverse 一隅，在 Project Hazelita 社群里共享着唯一一些自由的气息。和零音闹掰的朋友越来越多——零音甚至并不知道还应不应该称他们为「朋友」——偶尔零音也会自责吧。零音一方面怪罪于他们自身的问题，另一方面又自责于自己无法妥善处理好这些人际关系；而现在的零音，又要自责于，为何让别人自责于别人无法处理好零音与和零音已经闹掰的朋友之间的人际关系了。责怪是一条链子，于是拖垮了零音。无数人来过，在这里审视了那些曾经带给零音无数痛苦和绝望的东西，他们并不轻轻地走；他们却反而留下一些继续言语，所谓「我不想了解整件事情」、「我也不想带有偏见的看法」、「我只是觉得你们没有什么矛盾」、「我觉得你们本应和好」。Pwn 题 VOCAEND 的故事已经结束，hzlt!Game 也已结束半年。正如 VOCAEND 中所说的，歌曲会消逝；`data/` 中的一些歌曲，也已经消逝了。一部分缘由便是如此。这些话略显刺耳，但零音仍然要说：伤痕并不可逆。它可以康复，但一定会留下难看的疤痕。零音并不期望伤痛，但同时也不期望任何人无视零音的伤痛。况且，为无所谓的故人去损害一段现有的关系，本就是不可理喻的事情。你究竟是更乐意于立足现在，还是更想要回忆过去呢？这样的问题，零音不常问别人，可真正希望部分人认真思量一下吧。而零音也将继续作为史官——不必为正义或邪恶的史官——去记录发生过的一切。零音是其自己的史官。

说回宿命。零音为何离开，又为何不选择到来。

零音曾幻想过这样一片自由的天空——大家在广袤的天空之下畅所欲言，共同交流、共同进步。可真正在零音的一切美好期待过后迎接零音的，却是一个半军事化、半绩效制、半家长制的小型组织机器。零音并不想明说，但也不想避讳些什么了。没错，「职位不是权利(力)，而是责任」，但几乎所有关键机制都在把权力集中在少数人手里；人们当然知道全民投票的坏处，而事实上 Hazelita 社群也几乎因为全民投票的试运营而面临分崩离析的危害，但制衡、申诉、回避、透明记录、问责机制，也似乎一概不存在。而在这片已然混沌的灰色天空之上，竟然还存在着无限期高人一等的 [REDACTED]，祂被从同属于天空之下的人异化到了一个监督、冒着风险去催促、足够狠下心去决策的持续施压的诡谲的机器。天空之下，似乎没有人在意每个个体应当如何更好地成长了；所有人在意的，只是如何让人们持续产出、持续活跃、持续服从。而这微妙的差异，伴着刺鼻的等级气味、分类体制，人便不再是具体的人，而是一种管理状态了。

他们固然批评摆烂的人，却将这种批评上升为了道德债务，以至于成为了一种情感绑架：倘若你曾驻足于此，获得过这里的一草一木，你便已经欠这里东西了；如果你不继续付出，那便是忘恩负义。可这靠技术热爱才凝聚起来的组织啊，终究不是宗族、也不是公司、更不是精神债务收割机。人们往来于此，走走停停，固然可以感谢这里的一花一草、留恋这里的一枝一木；但因退出、失去热情和阶段性不活跃就被自动道德化，固然也不够健康吧。

至少零音不愿成为任何集体的燃料。

「稳定」「贡献」「活跃度」「清退」「督促」「催促」「负责人」「背锅人」。这些词构成了一个隐形的管理世界观：这里是一个需要被维护的机器，成员是机器上的零件；零件要么输出，要么被换掉。而这里的一系列规范，却又无时无刻不在给人一种隐形的压迫感——很多人完全意识不到，一个组织的章程，其语言会反过来塑造组织的气质。这种压迫感或许会默许这样的冷酷和羞辱是被许可的，甚至是——成熟的。对，那些在这里已颇有资历的学长学姐，即使已经离开了这片天空之下，却又将自己离开的经纬度转换为了高度，天然地赋予了他们自己一些高人一等的感觉，展现出一种莫名其妙的优越感。老生可以随时随地压力新人，将这片天空之下本身的问题，如烫手的山芋一般一层一层扔下去；若是这样的施压没有落到成效，反而给了他们更多自由发挥其言语的空间，将未能压力成功转换为新生不努力、无成效、不奋斗之类的各种言辞。对，管理层拥有解释权、老成员拥有经验权、高年级拥有监督权，新人需要被督促、被统计、被筛选、被证明。

唯一没有被当作自由的个体。

然后更多的人来到了这天空之下。他们当然会感觉自己不是被欢迎加入了一个共同体，而是被投放进一个评价体系。而对「技术」和「管理」的关系，却又构成了一种更深的矛盾。一方面，「我们是一个技术性的团体」，所以负责人需要有技术能力；另一方面，这里又的确欠缺管理、文书、宣传、活动组织能力。但这里给出的解决方案仍然是：技术部的人顺手做这些；或者非技术人员也参与日常学习活动以获得参与感。那结果不仍然是所有价值被技术中心主义吸走了吗？从而技术强的人被迫管理；管理强但技术不强的人得不到真正的尊重。这样下去，人们越来越相信一种单一价值了——「你技术强！你活跃！你贡献多！所以你更有资格说话！」而这恰好是一个技术组织最容易腐烂的路径。老成员将自己的积累当作合法的优越感；从而新人不再是伙伴而是下一批 **需要被筛选的材料** 。零音并不是说上文中提到的章程没有善意——没错，它太想解决问题了，太想让这片天空不再暗淡无光，太想让这里的一切传承下去了。但它的善意逐渐变成了控制欲——它想拯救这里，但它把天空放在了个体之前。

零音曾栖息于这样一片有些灰暗的天空之下——在这里遇到了三个人。

第一个人险些将零音从这天空的庇护中推了出去。祂坦言这里是一片圣地，祂不愿让外界的腐朽、不良的风气、矛盾和纠葛陷入这一片圣地，于是自己开起巨大的伞，为这片天空遮风挡雨。老实说祂做得很好；但却似乎仅限于很好。为祂心中这种神圣而不可侵犯的理想，祂甚至不惜损害自己的名声——固然，后世的人们都赞颂祂可歌可泣的事迹——谁还记得那些祂与零音仍素未谋面的日子里，祂如何尝试将零音也推出这片天空之下呢？只是因为零音「不好控制」。哎，控制欲！可怕的控制欲！得当的控制造就美好的集体，不当的控制毁掉这里的一切！

第二个人将不属于这里的超能力带了过来。在人与人脑力抗衡的赛场上，祂开着硅基生命铸就的无数个奇迹般的赛博生命，在这里「神挡杀神，佛挡杀佛」。一血！二血！排名不断靠前！一道题刚刚发出十分钟，Writeup 已经全部写完了！那本不是属于这片竞技场上的东西……不。那是赛场上本应出现的东西。技术在进步，我们也固然不愿开时代的倒车啊。这超人般的力量，我们曾也拥有过——零音靠着这力量登上了神坛，却又为何再次反对这力量的到来呢？

第三个人离开了这里。甚至早于零音的离开。祂曾与零音并肩奔跑过——在那片赛场上，那些初出茅庐的新人们，跑得比谁都要快。可惜祂愈跑愈急，愈跑愈快，跑得太快了，以至于将影子都甩在了身后。技与术在祂眼中愈来愈大，人却愈来愈小了；终于有一天，人渺小到了看不见的地步。零音曾试图喊住祂，祂回头看了一眼，又继续向前跑去了。零音也与祂闹过矛盾——即使祂全责——但祂如今与没事人一样。祂功利地选择了另一条赛道，与曾经并肩的人断绝了关系，临走前没有留下一句话。零音不该把祂当作是人的。本来就不该将祂当作是人类的。祂已是一台机器了。

混沌！混沌！无尽的混沌！为什么，为什么零音在一切伊始之时所做出的美好的期盼，最终都沦落为这样的下场？是零音自己的问题吗？是零音过于善良、过于真诚、过于相信一切都有其自然规律、过于肯定自己和他人的力量、过于相信善有善报、相信只要情真意切就可以挽回一切、相信自己具有能够改变自己和身边所有人的力量了吗？为什么？为什么？究竟是为什么？为什么，为什么这一切的悲痛和绝望最终都落回到零音的身上？为什么明明不是零音的过错，却需要零音也承担一份结果？明明这一切都不应当出错的；明明这并不是零音所想的预期结果的。那么，究竟是谁在捣鬼？谁要为这件事情负责？谁会为零音撑腰？谁在热忱地热爱零音？谁在匿名地嫉妒零音？谁在背后为零音悄悄使下无尽的绊子？谁顶着偏执的爱的名义就可以不理会零音的呼声？谁靠着认识和熟悉的借口就可以抹杀零音的诉求？说话。说话！为什么没有人说话呢！为什么，为什么到这种时刻，就没有人挺身而出了呢？为什么到这种时刻，一切的过错就可以被原谅、一切的责任就可以被免去、一切的记忆就可以被消除；一切的痛苦、一切的失望、一切的抗争，就都沦为了笑话、沦为了无理取闹、沦为了无谓的挣扎呢？这世间，究竟是谁在主持公道呢？为什么，受伤的总是老实人呢？

一切，会结束吗？

但好在零音已经过了自己的 19 岁生日。这一切的抗争，或许终将迎来结束吧。

倘若零音能更有勇气的面对前途未知的艰难险阻。倘若零音能够在自己遭遇伤害的时候更加挺身而出。

或许结局会变好的。

未来会好吗？零音并不确信于这一点。

但，零音已远去这片阴沉的天空。走出这片乌云密布的天空，外面似乎并未如何下雨。

## 💻 Build

Requirements: GCC, make, libc, libm.

```sh
make
```

Output binary: `data/vocaend`

## 🧪 Run (local)

The binary expects `song*.txt` to exist in the current working directory and
loads the flag from a local `flag` file.

```sh
echo 'flag{test}' > data/flag
cd data
./vocaend
```

To skip lyric delays:

```sh
VOCA_FAST=1 ./vocaend
```

## 🔧 Solve Path

The intended archived solve path is documented in `WRITEUP.md`. A local
regression helper is also available:

```sh
make
tools/solve_local.py
```

The helper uses a PTY and sends inputs step by step. Piping the full transcript
at once is not equivalent to the original interaction because the challenge uses
raw `read(2)` calls.

## 🚢 Legacy Docker

The original competition deployment has been moved to `legacy/Dockerfile`. It is
kept for archival reference and is no longer the recommended local run path:

```sh
docker build -f legacy/Dockerfile -t vocaend .
docker run --rm -p 9999:9999 -e GZCTF_FLAG=flag{test} vocaend
```

## 📁 Layout

- `src/` challenge source code
- `inc/` focused headers and lyric macros
- `data/` song text files and runtime artifacts
- `build/` build intermediates
- `legacy/` original competition deployment files
- `tools/` local archival verification helpers
